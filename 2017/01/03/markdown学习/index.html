    markdown的学习笔记，包括斜体，粗体，标题，链接，表格，块引用等语法。

    markdown语法

    斜体

    在文字两边加上下划线_或星号*
    示例：

    _not_
*not*

    呈现：
    not
    not

    粗体

    在文字两边加上两个星号**
    示例：

    **not**

    呈现：
    not
    注：以上两个可以跨行使用。
    同时加粗和斜体
    示例：

    **_not_**

    呈现：
    not

    删除线

    示例：

    ~~not~~

    呈现：
    not

    标题

    在文字左边加上#，几级标题前面就是几个#。
    一级标题#not
    三级标题###not

    注：不能给标题加粗，但是可以用斜体来强调其中的某些单词。

    链接

    行内连接:[]()
    示例： 

    [Visit GitHub!](www.github.com)

    呈现：
    Visit GitHub!
    悬停在链接上，有一个工具提示：

    [Upstage](https://github.com/upstage/ "Visit Upstage!")

    参考连接
    另一种链接类型称为参考链接。顾名思义，链接实际上是引用文档中的另一个位置。参考链接风格的优点是到同一地点的多个链接仅需要被更新一次。
    示例：

    Here's [a link to something else][another place].
Here's [yet another link][another-link].
And now back to [the first link][another place].

[another place]: www.github.com
[another-link]: www.google.com

    命名锚点

    命名锚点使您能够跳到同一页面上的指定锚点。
    示例：

    #### Table of Contents
  * [Chapter 1](#chapter-1)
  * [Chapter 2](#chapter-2)
  * [Chapter 3](#chapter-3)
将会跳转到如下章节:
#### Chapter 1 <a id="chapter-1"></a>
Content for chapter one.
#### Chapter 2 <a id="chapter-2"></a>
Content for chapter one.
#### Chapter 3 <a id="chapter-3"></a>
Content for chapter one.

    呈现：

    Table of Contents

    • Chapter 1
    • Chapter 2
    • Chapter 3
      将会跳转到如下章节:

      Chapter 1

      Content for chapter one.

      Chapter 2

      Content for chapter one.

      Chapter 3

      Content for chapter one.

    引用

    引用是一个被特别格式化以引起读者注意的句子或段落。
    示例：

    >not

    呈现：

    not

    您还可以在块引用的每一行上插入一个插入符号。 如果您的引用跨越多个段落，这是特别有用。请注意，即使空行必须包含插入符号。 这确保整个块引用组合在一起。

    > His words seemed to have struck some deep chord in his own nature. Had he spoken
of himself, of himself as he was or wished to be? Stephen watched his face for some
moments in silence. A cold sadness was there. He had spoken of himself, of his own
loneliness which he feared.
>
> —Of whom are you speaking? Stephen asked at length.
>
> Cranly did not answer.

    块引用也可以嵌套，例如：

    > Donec massa lacus, ultricies a ullamcorper in, fermentum sed augue.
Nunc augue augue, aliquam non hendrerit ac, commodo vel nisi.
>> Sed adipiscing elit vitae augue consectetur a gravida nunc vehicula. Donec auctor
odio non est accumsan facilisis. Aliquam id turpis in dolor tincidunt mollis ac eu diam.

    呈现：

    Donec massa lacus, ultricies a ullamcorper in, fermentum sed augue.
    Nunc augue augue, aliquam non hendrerit ac, commodo vel nisi.

    Sed adipiscing elit vitae augue consectetur a gravida nunc vehicula. Donec auctor
    odio non est accumsan facilisis. Aliquam id turpis in dolor tincidunt mollis ac eu diam.

    列表

    无序列表
    示例： 

    * Milk  
* Eggs  
* Salmon   
* Butter

    呈现：

    • Milk
    • Eggs
    • Salmon
    • Butter

    您可以使用以下任何符号来表示每个列表项的项目符号：
    示例： 

    * valid bullet
- valid bullet
+ valid bullet

    有序列表
    示例： 

    1. Crack three eggs over a bowl
2. Pour a gallon of milk into the bowl
3. Rub the salmon vigorously with butter
4. Drop the salmon into the egg-milk bowl

    呈现：

    1. Crack three eggs over a bowl
    2. Pour a gallon of milk into the bowl
    3. Rub the salmon vigorously with butter
    4. Drop the salmon into the egg-milk bowl

    提示：如果每个数字都只是使用1.表示，Markdown将自动编号每个项目。
    有时，您可能会发现需要将一个列表嵌套在另一个列表中。 不要恐惧，因为Markdown语法是完全一样的。

    你需要做的是记住缩进每个星号一个空格多于前面的项目。

    假设您想创建一个需要一些附加上下文（但不是另一个列表）的项目符号列表。您的段落必须在项目符号下面的所有行上开始，并且必须缩进至少一个空格。

    段落

    每个段落后面有一个空行，或者在段落的结尾加上两个空格。

    类似html的hr标签，在markdown中的表示如下：

    ___：三个连续的下划线
---：三个连续的破折号
***：三个连续的星号

    代码

    在文字两边加上`，或者通过至少四个空格缩进几行代码。
    使用```来标示多行代码。
    代码的语法高亮，直接在```后面加上所使用的语言的扩展名，例如js：
    ```js
    function()
    {
    }
    ```

    呈现：

    提示：要使语法高亮起作用，需要安装并启用Highlight插件。 它反过来利用一个jQuery插件，所以jquery需要加载到你的主题。

    表格

    通过在每个单元格之间添加管道作为分隔符，并通过在标题下面添加一行破折号（也用条形分隔）来创建表。 注意，管道不需要垂直对准。
    注：切记表格要与上面的文本内容空一行。否则解析不出来.

    示例：
    | Option | Description |
    | —— | ———– |
    | data | path to data files to supply the data that will be passed into templates. |
    | engine | engine to be used for processing templates. Handlebars is the default. |
    | ext | extension to be used for dest files. |

    呈现：

    Option Description
    data path to data files to supply the data that will be passed into templates.
    engine engine to be used for processing templates. Handlebars is the default.
    ext extension to be used for dest files.

    在任何标题下面的短划线右侧添加冒号将会右对齐该列的文本。
    示例：
    | Option | Description |
    | ——:| ———–:|
    | data | path to data files to supply the data that will be passed into templates. |
    | engine | engine to be used for processing templates. Handlebars is the default. |
    | ext | extension to be used for dest files. |

    呈现：

    Option Description
    data path to data files to supply the data that will be passed into templates.
    engine engine to be used for processing templates. Handlebars is the default.
    ext extension to be used for dest files.

    转义字符

    Markdown中的转义字符为\，转义的有：
\\ 反斜杠
\` 反引号
\* 星号
\_ 下划线
\{\} 大括号
\[\] 中括号
\(\) 小括号
\# 井号
\+ 加号
\- 减号
\. 英文句号
\! 感叹号
